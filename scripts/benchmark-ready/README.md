# Benchmark Ground Truth

## Структура
- `train/` — 8 файлов для итераций и настройки парсера
- `holdout/` — 4 файла для финальной валидации (НЕ оптимизировать под них)

## Правила
- Правки парсинга делаются по результатам на train/
- holdout/ прогоняется каждые 3–4 правки для проверки обобщения
- Если holdout упал на >3% — правка обобщается плохо, пересмотреть подход
