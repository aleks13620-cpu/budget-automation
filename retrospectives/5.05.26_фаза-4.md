## Что пошло не так / неожиданности

- Агент не запустил скрипт (ограничение промпта "трогать только scripts/matching-baseline.py") — JSON пришлось генерировать и коммитить отдельно оркестратору
- Коммит JSON (536bd29) оказался лишним — следующим коммитом (5cf4be2) файл был untracked. В git history остался артефакт с project names
- Первоначальная оценка оркестратора по hardening была "overkill" — агент убедил фактами проекта, оркестратор пересмотрел

## Технический долг, выявленный в фазе

- matching-baseline.json коммит 536bd29 содержит project names в git history — `git filter-branch` / BFG не проводился (приватный репо, низкий риск)
- PRB-001 (transient artifacts в git) частично закрыт для этого файла, но проблема системная
- match_rate 0.1% — матчер работает, но правила почти не наработаны (6 matches на 12 проектов)

## Правила, выведенные из фазы

- Generated artifacts (JSON/CSV из БД) не трекать в git — добавлять в .gitignore сразу при создании скрипта
- Промпт для агента: разрешать создание output-файлов явно, либо запускать скрипт отдельно
- Security review агента проверять по фактам проекта (docker-compose, connection.ts, problem-registry) перед оценкой "overkill"
