## Что пошло хорошо

- Все 5 шагов прошли с первой попытки, без fix-промптов
- git diff на каждом шаге — чистый, только целевые файлы
- Codex правильно не коммитил docs-изменения, которые были вне скоупа
- Gemini 2.5 Flash сразу вышел выше всех порогов: модель готова к интеграции

## Что пошло не так / неожиданности

- В рабочем дереве с самого начала висели pre-existing docs-изменения (5 устаревших планов в active/). Агенты их не трогали, но они создавали шум в `git status` весь проект
- `plans:check:strict` падал с exit 1 из-за 6 активных планов — обнаружили только при попытке пуша после фазы 2
- `OPENROUTER_API_KEY` не был доступен в окружении агента — потребовалось создавать `.env` вручную

## Технический долг, выявленный в фазе

- Docs-хаос в `docs/plans/active/` — решён в конце фазы 2 отдельным коммитом
- `.env` не документирован: нет `.env.example` с нужными ключами — запланировано в шаге 3а

## Правила, выведенные из фазы

- Перед первым push проверять `npm run plans:check:strict`
- API-ключи описывать в `.env.example` сразу при добавлении нового сервиса
- Diff-верификация обязательна: только файлы из промпта → PASS, лишнее → FAIL
