# Руководство пользователя: Обучение системы сопоставления

## Введение

Система автоматически сопоставляет позиции спецификации с позициями счетов поставщиков. Чем больше примеров она видит — тем лучше работает автоматическое сопоставление. Это руководство объясняет, как обучить систему на уже готовых материалах.

---

## Способ 1: Импорт готовых эталонных матчей из Excel

Это **самый быстрый способ** обучить систему — если у вас уже есть таблица с правильными сопоставлениями из прошлых проектов.

### Что подготовить

Excel-файл (.xlsx или .xls) с тремя колонками (названия колонок — произвольные, система определяет их автоматически по ключевым словам):

| Наименование в спецификации | Наименование в счёте | Поставщик (опционально) |
|---|---|---|
| Задвижка стальная DN50 | Задвижка DN50 фланцевая | ООО "Арматура" |
| Кран шаровой DN25 | Кран ш/з Ду25 | — |

**Требования к колонкам:**
- Колонка спецификации: содержит слова `спецификация`, `spec`, `наименование спец` или `позиция`
- Колонка счёта: содержит слова `счёт`, `invoice`, `наименование счёт` или `товар`
- Колонка поставщика (необязательна): содержит слово `поставщик` или `supplier`

### Как импортировать

1. Откройте проект
2. В разделе **«Обучение системы»** нажмите **«Выбрать файл»**
3. Выберите подготовленный Excel-файл
4. Нажмите **«Импортировать эталонные матчи»**
5. Система покажет результат: `Импортировано N правил`

После импорта — запустите сопоставление. Правила применятся с уверенностью **0.95** (Tier 2 — «Правила»).

## Способ 2: Обучение в процессе работы (подтверждение / отклонение)

Система учится на каждом вашем действии в режиме сопоставления.

### Подтверждение матча

Когда вы нажимаете **«✓ Подтвердить»** рядом с предложенным сопоставлением:
- Система сохраняет эту пару как правило сопоставления
- При следующем запуске похожие позиции будут сопоставлены автоматически
- Действие записывается в журнал обучения (страница «📊 Обучение»)
- При уверенности матча **не ниже 0,85** система может **автоматически пополнить словарь строительных сокращений**: если в одном названии есть короткое обозначение, а в другом — развёрнутая форма, пара добавляется для будущих сопоставлений (консервативный алгоритм, без перезаписи ваших Excel-правил)

### Автоматическое запоминание режима GigaChat для поставщика

Если счёт впервые успешно разобран через **GigaChat** (не менее трёх позиций) и для поставщика ещё **нет** сохранённой схемы колонок — система сохраняет признак «удобно тянуть через GigaChat». При следующих счетах этого поставщика ИИ получает усиленный контекст (без подмены уже настроенного вручную маппинга Excel).

### Отклонение матча

Когда вы нажимаете **«✗ Удалить»** для **неподтверждённого** предложения:
- Создаётся **негативное правило** — эта пара больше никогда не будет предложена
- Система запоминает ошибку и не повторяет её
- Действие записывается в журнал обучения

### Ручное сопоставление (М2/М3)

При ручном указании правильной пары через кнопку «+ Сопоставить вручную»:
- Создаётся правило с уверенностью 1.0
- Записывается в журнал как `manual_select`

## Способ 3: Проверка через GigaChat (пограничные кандидаты)

Для позиций, где система «не уверена» (уверенность 25–40%), можно запросить проверку через GigaChat.

### Как использовать

1. Откройте вкладку **«Сопоставление»**
2. Нажмите кнопку **«🤖 Проверить (GigaChat)»**
3. Система проверит до 5 пограничных пар за раз
4. При положительном ответе GigaChat — уверенность поднимается до 0.7 (матч сохраняется)
5. При отрицательном ответе — кандидат удаляется

Результаты кэшируются — повторных запросов для уже проверенных пар не будет.

> **Требование:** GigaChat должен быть настроен (переменные `GIGACHAT_*` в `.env`)

## Как проверить качество обучения

### Прогресс-бар в проекте

На странице проекта в разделе «Сопоставление» отображается:
```
45 / 120 подтверждено (37%)
[████░░░░░░░░░░░░░░░░]
```

**Цвет полосы:**
- 🔴 Красный — менее 30% покрытия
- 🟡 Жёлтый — 30–70%
- 🟢 Зелёный — более 70%

### Детализация в режиме сопоставления

В разделе «Сопоставление» показывается разбивка по тирам:
```
Артикул: 12 | Правила: 34 | Схожесть: 18 | Хар-ки: 5 | Вручную: 3
```

- **Артикул** — точное совпадение артикула (уверенность 0.98) — лучший результат
- **Правила** — применено сохранённое правило (из импорта или обучения) — хороший результат
- **Схожесть** — совпадение по названию (≥ 0.7) — требует проверки
- **Хар-ки** — совпадение с учётом характеристик (≥ 0.5) — требует проверки
- **Вручную** — сопоставлено оператором

### Журнал обучения (страница «📊 Обучение»)

Кнопка в верхнем меню → показывает все действия оператора:
- Сколько матчей подтверждено
- Сколько отклонено
- Сколько сопоставлено вручную
- Временная шкала активности

## Рекомендуемый порядок обучения для нового проекта

### Шаг 1: Импорт эталона (если есть готовые данные)
Подготовьте Excel с правильными сопоставлениями из аналогичных прошлых проектов.
Импортируйте через «Обучение системы» в карточке проекта.
**Ожидаемый результат:** 60–80% позиций сопоставятся автоматически.

### Шаг 2: Первый запуск сопоставления
Нажмите «Запустить сопоставление» (кнопка на странице сопоставления).
Просмотрите предложенные матчи.

### Шаг 3: Проверка и обучение
- Подтвердите правильные сопоставления (кнопка ✓)
- Отклоните неправильные (кнопка ✗) — система запомнит ошибки
- Сопоставьте вручную то, что система не нашла

### Шаг 4: Повторный запуск
После подтверждения 20–30 матчей — запустите сопоставление снова.
Правила применятся к оставшимся позициям.

### Шаг 5: GigaChat для «трудных» позиций
Нажмите «🤖 Проверить (GigaChat)» для проверки пограничных кандидатов.

## Советы

**Как получить максимум от импорта:**
- Используйте счёт и спецификацию из того же проекта или максимально похожего
- Чем точнее наименования в таблице совпадают с реальными позициями — тем выше точность

**Что делать с позицией «без матча»:**
- Проверьте, не является ли это нестандартной позицией (доставка, монтаж)
- Если позиция есть в счёте под другим названием — сопоставьте вручную, система запомнит

**Как очистить ошибочные правила:**
- Ошибочные автоматические матчи отклоняйте (✗) — создаётся негативное правило
- Это надёжнее удаления, так как предотвращает повторное появление

**Периодичность переобучения:**
- После каждого нового проекта — импортируйте эталон для пополнения базы правил
- Правила накапливаются глобально и применяются ко всем проектам
