# Исследование рынка — TL;DR

_Синтез по 5 углам исследования. Данные: training knowledge, cutoff август 2025._

---

## Что нашли за одну строчку

**Западный рынок автоматизирует procurement workflow. Никто — ни на Западе, ни в России — не решил задачу fuzzy-matching строительной спецификации со счётами поставщиков с обучением из подтверждений.** Budget Automation занимает незанятую нишу.

---

## Главные находки

### 1. Рынок есть и он финансируется
Строительный procurement привлекает серьёзные деньги:
- **Kojo** (материалы для подрядчиков) — $50M+ Series C, Battery Ventures
- **Constrafor** (субподряд + compliance) — $106M Series B, Bessemer + Spark Capital
- **Agora** (Израиль, сравнение цен материалов) — $3M seed, Bessemer
- **Trunk Tools** (AI для PM) — $25M Series A, Sequoia
- **Buildxact** (ближайший функциональный аналог) — $199–599/мес, австралийский SaaS

a16z, Sequoia, Bessemer активны в пространстве. Brick & Mortar Ventures — специализированный фонд именно под construction tech.

### 2. Никто не решил core задачу
Ни один из ~20 проверенных продуктов не умеет:
- принять произвольный Excel-спецификации (не normalized)
- принять PDF-счёт поставщика
- автоматически сопоставить строки fuzzy-matching + AI
- обучаться на подтверждениях оператора

Ближайший по намерению — **Buildxact** (загрузка PDF-прайса), но там базовый OCR без интеллекта. **Nanonets** умеет extraction + PO-matching, но PO должен быть structured и не строительный.

### 3. Российский рынок — белое пятно
**ГРАНД-Смета** и **Турбо-Смета** — доминирующие инструменты создания смет, в них нет модуля сравнения с поставщиками. **1С:Управление строительством** требует чистые артикулы в каталоге — не PDF. После ухода Autodesk в 2022 российский рынок ещё более изолирован. Зарубежные игроки сюда не идут.

### 4. Провалившиеся стартапы учат ошибкам
- **Alice Technologies** (~$30M) — закрылась 2023. Enterprise-only + 12-18 месячный цикл продаж.
- **Constrafor** — реструктурировалась: росла быстрее, чем рынок готов платить.
- Несколько "smart BOM" стартапов 2019–2022 в solar/construction — все развернулись или закрылись. Причина: стоимость нормализации данных оказалась выше ожидаемой, нет сетевого эффекта.

### 5. Open source заменяет 6+ месяцев разработки
Стек для document parsing + fuzzy matching зрелый: **Docling** (IBM, PDF-таблицы), **pdfplumber**, **Unstructured.io**, **RapidFuzz**, **Splink** (вероятностное связывание). Всё MIT/Apache, форкабельно.

## Главная дельта идеи

**Budget Automation = единственный продукт, который берёт строительную спецификацию и счёт поставщика "как есть" и выдаёт сопоставление, обучаясь при этом.** На российском рынке — прямых конкурентов нет вообще. На западном — прямых тоже нет, есть соседи по квадранту.

Уникальность стоит на трёх столпах:
1. **Fuzzy matching с самообучением** (никто не делает)
2. **Русскоязычный строительный домен** + GigaChat (data sovereignty для госконтрактов)
3. **Работа с форматами "как есть"** — без преднастройки колонок и артикулов
