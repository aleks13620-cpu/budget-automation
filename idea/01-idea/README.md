# Идея проекта

## Суть в одном абзаце

Строительный проект рождает два документа: **спецификация** (что закупить, составлена проектировщиком) и **счета/прайсы поставщиков** (что продаётся и по какой цене). Между ними — ручная сверка, которая занимает часы и в которой легко ошибиться: один и тот же кабель у разных поставщиков называется по-разному, единицы измерения расходятся, артикулы присутствуют только иногда. Budget Automation автоматизирует именно эту сверку: загружаешь спецификацию и пачку счетов, система сама сопоставляет позиции и предлагает оператору только спорные случаи на подтверждение.

## Что система делает

1. **Импортирует спецификацию** из Excel-файла — распознаёт структуру таблицы (заголовок, разделы, строки позиций) без ручной настройки колонок.
2. **Импортирует счета поставщиков** из PDF и Excel — извлекает таблицы с позициями товаров, ценами, количеством.
3. **Сопоставляет позиции** по четырём уровням: точный артикул → выученные правила → нечёткое совпадение по названию → название + характеристики.
4. **Показывает оператору** неуверенные совпадения, принимает подтверждение/отклонение.
5. **Запоминает** подтверждённые совпадения как правила — при следующей загрузке от того же поставщика они применяются автоматически.
6. **Формирует итоговую спецификацию** в Excel с ценами из выбранных счетов.

## Что делает систему нетривиальной

- Счета приходят в разных форматах (скан PDF, сложная Excel с объединёнными ячейками, таблица без заголовков) — система сначала пробует классический парсер, при неудаче передаёт документ в GigaChat.
- Цены бывают с НДС и без — система хранит ставку НДС на уровне поставщика и нормализует цены при сравнении.
- Наименования у разных поставщиков одной позиции могут отличаться кардинально — система учится на подтверждённых парах и строит словарь синонимов.
- Спецификации делятся на разделы (электрика, вентиляция, водоснабжение и т.д.) — система определяет раздел автоматически по ключевым словам в заголовке и самих позициях.
