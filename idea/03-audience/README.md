# Аудитория

## Кто использует систему — извлекается из кода

### Оператор (основной пользователь)
Судя по `OPERATOR_GUIDE.md` и наличию `operator_feedback` в базе, это человек, который:
- Загружает документы проекта (спецификации и счета)
- Проверяет и подтверждает результаты парсинга
- Принимает решения по спорным совпадениям
- Фиксирует ошибки системы через форму обратной связи

В строительном контексте — вероятно, **сметчик** или **специалист по снабжению/закупкам**.

### Администратор / куратор
Судя по `GlobalFeedbackPage` и `DEPLOY.md`:
- Видит все ошибки по всем проектам
- Управляет развёртыванием на продакшн-сервере

## Что НЕ извлекается из кода

- Размер организации-пользователя (один человек, отдел или несколько компаний)
- Есть ли разделение ролей (пока один пользователь на всё — нет аутентификации)
- Частота использования (сколько проектов в месяц, сколько счетов в проекте)
- Болевые точки, которые привели к созданию — не в коде
