## Компания
Строительная компания, монтаж инженерных сетей на объектах >5 000 м².
Разделы: отопление, вентиляция, ВК, ИТП, автоматизация, электрика, слаботочка.
~50 человек, 5–6 площадок одновременно, средний контракт ~40 млн руб. / 6–8 мес.

## Ключевая боль
1 расчёт = 1–1.5 недели, 3 человека, потолок 4–5 проектов/мес.
90% расчётов не конвертируются. Увеличение штата нерентабельно.

## Текущий процесс (as-is)
1. PDF-проект → спецификация переводится в Excel вручную
2. Снабжение рассылает запросы ~20 поставщикам по почте
3. Поставщики присылают счета (PDF + Excel) — цены проставляют вручную
4. Сметчик добавляет стоимость работ → итог → КП

## Продукт
Веб-система: автоматически сопоставляет позиции спецификации со счетами поставщиков,
проставляет цены, формирует отчёт по разделам для бюджета.

Стек: Node.js + TypeScript, SQLite, GigaChat-2 (парсинг), React + Vite, Timeweb.

## Целевые KPI после MVP

| Метрика | До | После |
|---|---|---|
| Время расчёта | 7–10 дней | 1–2 дня |
| Людей | 3 | 1 |
| Проектов/мес | 4–5 | 15–20 |
| Стоимость расчёта | ~40–50 тыс. руб. | ~8–10 тыс. руб. |

Точность автосопоставления: ≥80–85%, цель после обучения: 95%+.

## Роли

| Роль | Кто | Задача |
|---|---|---|
| Владелец / заказчик | Артём | Итог по разделам → вставляет в бюджет → выпускает КП |
| Снабжение | Иван + Сергей | Работа со спецификацией, контроль счетов |
| Оператор | Студент/помощник | Загрузка счетов, подтверждение сопоставлений |

## Этапы разработки
- **MVP (Этап 1):** расчёт затратной части по материалам (текущий этап)
- **Этап 2:** закупка, логистика, склад, объектный учёт
- **Этап 3+:** интеграция с AMO CRM, 1С, мобильное приложение
