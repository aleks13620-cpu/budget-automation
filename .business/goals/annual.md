# Годовые цели

## Цель для клиента (Артём)
Снять потолок по количеству проектов. Перейти с 4–5 проектов/мес на 15–20+ без пропорционального роста штата. Масштабироваться в регионы, нанять больше менеджеров по продажам.

**Узлы, которые нужно развязать:**
1. Расчёт материалов (MVP — текущий этап)
2. Объектный учёт: заявки → поставки → выполненные работы → акты (Этап 2, Артём считает даже важнее)

## Цель для разработчика (Алексей)
Сдать работающий MVP → запустить обучение на реальном проекте («Новая Самара») → масштабировать на другие строительные компании (потенциальный SaaS).

## Потенциал масштабирования
При решённом расчётном блоке Артём может:
- Нанять больше менеджеров по продажам
- Уйти в регионы
- Параллельно вести десятки проектов в расчёте
