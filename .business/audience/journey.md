# Путь пользователя (Customer Journey)

## Типичный день оператора (после внедрения MVP)

| Время | Действие |
|-------|----------|
| 9:00 | Получает новый проект от менеджера |
| 9:10 | Создаёт проект в системе, загружает спецификацию Excel |
| 9:15 | Снабжение прислало 5 счетов → загружает в систему |
| 9:20 | Система обработала файлы, предложила сопоставления |
| 9:30 | Подтверждает зелёные совпадения одним кликом |
| 10:00 | Проверяет жёлтые совпадения, отклоняет неверные |
| 10:30 | Назначает разделы позициям (система предложила варианты) |
| 11:00 | Открывает отчёт, проверяет итоги, скачивает Excel |
| 11:05 | Отправляет файл менеджеру для подготовки КП |

**Итого: ~2 часа вместо 7–10 дней**

## Путь до внедрения (как было)
1. Получить PDF проекта
2. Вручную перебить спецификацию в Excel
3. Разослать запросы ~20 поставщикам по почте
4. Собирать ответы (PDF/Excel счета)
5. Вручную проставлять цены построчно
6. Суммировать по разделам вручную
7. Перенести итоги в бюджет → КП

Время: **7–10 дней**, участвует **3 человека**.
