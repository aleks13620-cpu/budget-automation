# Безопасность

## Что реализовано

- **Ограничение типов файлов:** загрузка принимает только `.pdf`, `.xlsx`, `.xls`, `.jpg`, `.jpeg`, `.png`, `.tiff`, `.bmp`
- **Ограничение размера файла:** 50 MB (Multer)
- **Санитизация имён файлов:** спецсимволы заменяются перед сохранением (`fileUtils.ts`)
- **Foreign keys + CASCADE:** база не допускает висящих ссылок
- **WAL mode:** предотвращает корруцию базы при конкурентных записях
- **Переменные окружения:** секреты (GigaChat API key) хранятся в `.env`, не в коде

## Что НЕ реализовано

- **Аутентификация:** нет — любой, кто достучался до порта 3001, имеет полный доступ
- **Авторизация/роли:** нет — нет разделения прав
- **CORS ограничения:** `cors()` без параметров — разрешено всё; в продакшне нужно ограничить origins
- **Rate limiting:** нет
- **Шифрование базы:** SQLite-файл не зашифрован
- **HTTPS:** не настроено на уровне приложения (предположительно через reverse proxy или не используется)
- **Валидация входных данных:** частичная — нет систематической проверки тел запросов
- **Аудит действий:** нет лога кто/что/когда изменил (только history снимки)

## Текущая модель угроз

Система предполагает **доверенную сеть** — либо локальное использование, либо доступ через VPN. Это одиночный инстанс для одного или небольшого числа пользователей. Публичный доступ без аутентификации — риск.
