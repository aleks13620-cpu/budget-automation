# Интеграции

## GigaChat API (Сбер)

**Статус:** Подключена, активно используется

**Где используется:**
- Парсинг нечитаемых PDF-счетов (fallback когда классический парсер возвращает 0 позиций)
- Парсинг изображений счетов (JPG, PNG, TIFF, BMP)
- Обогащение спецификаций (дозаполнение полей: характеристики, производитель, артикул)
- Валидация слабых совпадений (Tier 5)

**Конфигурация (`.env`):**
```
GIGACHAT_AUTH_KEY=<base64-encoded OAuth key>
GIGACHAT_SCOPE=GIGACHAT_API_PERS
GIGACHAT_MODELS_FILES=GigaChat-2-Lite,GigaChat-2-Pro,GigaChat-2-Max
```

**Стратегия:** lite-first с эскалацией качества — сначала пробует Lite, при низком качестве результата повторяет с Pro/Max.

**Кэш:** результаты сохраняются в `gigachat_file_cache` по SHA256-хэшу файла. Повторная загрузка того же файла не делает API-запрос.

**Сервисы:**
- `gigachatService.ts` — аутентификация (OAuth токен с TTL), вызовы API
- `gigachatParser.ts` — парсинг счетов
- `gigachatSpecParser.ts` — обогащение спецификаций
- `gigachatParseQuality.ts` / `gigachatSpecParseQuality.ts` — оценка качества ответа

## Интеграции, которые НЕ реализованы

- Email (нет исходящей почты, нет уведомлений)
- 1C / ERP / учётные системы — не извлекается из кода
- Внешний API для сторонних клиентов — нет
- Webhook-уведомления — нет
- Облачное хранилище (S3/GCS) — файлы хранятся локально в `data/uploads/`
- Mistral API — ключ присутствует в `.env` (`MISTRAL_API_KEY`), но активного использования в коде не обнаружено
