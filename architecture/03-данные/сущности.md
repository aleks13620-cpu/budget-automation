# Сущности базы данных

## Основные сущности

### projects
Контейнер для всех данных одного строительного проекта.
- `id`, `name`, `description`, `created_at`, `updated_at`

### suppliers
Поставщик товаров. Создаётся автоматически из имени файла счёта при первой загрузке.
- `id`, `name`, `contact_info`, `vat_rate` (ставка НДС, %), `prices_include_vat` (0/1), `email`, `created_at`

### specifications
Документ спецификации по разделу. Один проект → несколько спецификаций (по разделам).
- `id`, `project_id`, `section` (название раздела), `file_name`, `raw_data` (JSON сырых данных), `parse_source` (`excel`/`gigachat`), `created_at`

### specification_items
Строка (позиция) из спецификации.
- `id`, `project_id`, `specification_id`, `position_number`, `name`, `characteristics`, `equipment_code`, `article`, `product_code`, `marking`, `type_size`, `manufacturer`, `unit`, `quantity`, `section`, `parent_item_id`, `full_name`, `created_at`

### invoices
Счёт (документ) поставщика.
- `id`, `project_id`, `supplier_id`, `invoice_number`, `invoice_date`, `file_name`, `file_path`, `total_amount`, `vat_amount`, `status`, `parsing_category` (A/B/C), `parsing_category_reason`, `discount_detected` (0/1), `needs_amount_review` (0/1), `vat_rate`, `prices_include_vat`, `is_manual` (0/1), `discount_applied`, `created_at`

### invoice_items
Строка (позиция) из счёта.
- `id`, `invoice_id`, `article`, `name`, `unit`, `quantity`, `quantity_packages`, `price`, `amount`, `row_index`, `is_delivery` (0/1), `is_manual` (0/1), `needs_unit_review` (0/1), `original_price`, `original_unit`, `created_at`

### matched_items
Результат сопоставления позиции спецификации с позицией счёта.
- `id`, `specification_item_id`, `invoice_item_id`, `confidence` (0–1), `match_type` (article/rule/fuzzy/name_chars), `is_confirmed` (0/1), `is_selected` (0/1), `is_analog` (0/1), `source`, `created_at`

## Конфигурации и обучение

### matching_rules
Выученные правила сопоставления из подтверждённых оператором пар.
- `id`, `specification_pattern`, `invoice_pattern`, `confidence`, `is_analog`, `is_negative`, `times_used`, `supplier_id`, `source`, `created_at`, `updated_at`

### supplier_parser_configs
Сохранённая конфигурация парсера для поставщика (маппинг колонок, строка заголовка).
- `id`, `supplier_id`, `config` (JSON), `created_at`, `updated_at`

### specification_parser_configs
Конфигурация парсера для конкретного файла спецификации.
- `id`, `specification_id`, `header_row`, `column_mapping` (JSON), `merge_multiline`, `created_at`, `updated_at`

### unit_conversion_triggers
Правила определения конвертации единиц измерения (глобальные).
- `id`, `keyword`, `from_unit`, `to_unit`, `description`, `created_at`

### construction_synonyms
Словарь аббревиатур строительной документации, накопленный из совпадений.
- `id`, `abbreviation`, `full_form`, `category`, `source`, `times_used`, `created_at`

### size_synonyms
Синонимы единиц измерения размеров.
- `id`, `canonical`, `synonym`

### spec_parse_rules
Правила исправлений из GigaChat-обогащения спецификаций.
- `id`, `specification_id`, `field`, `raw_value`, `corrected_value`, `times_used`, `created_at`

## Прайс-листы (вспомогательные)

### price_lists
Альтернативный прайс-лист поставщика (не основной счёт).
- `id`, `project_id`, `supplier_id`, `file_name`, `file_path`, `status`, `parser_config` (JSON), `created_at`

### price_list_items
Позиции из прайс-листа.
- `id`, `price_list_id`, `article`, `name`, `unit`, `price`, `row_index`, `created_at`

## Кэши

### gigachat_file_cache
Кэш результатов GigaChat по SHA256-хэшу файла. Избегает повторных API-запросов.
- `file_hash`, `purpose`, `response_json`, `created_at`

### gigachat_match_cache
Кэш результатов валидации совпадений через GigaChat.
- `id`, `spec_text`, `invoice_text`, `is_match`, `created_at`

## История версий

### invoice_items_history
Снимки состояния позиций счёта (для отката).
- `id`, `invoice_id`, `version`, `items_snapshot` (JSON), `action`, `created_at`

### specification_items_history
Снимки состояния позиций спецификации (для отката).
- `id`, `specification_id`, `version`, `items_snapshot` (JSON), `action`, `created_at`

## Обратная связь

### operator_feedback
Сообщения об ошибках от оператора.
- `id`, `type`, `project_id`, `spec_item_id`, `invoice_item_id`, `comment`, `status` (new/resolved), `created_at`
