# Технологический стек

## Backend

| Технология | Версия | Роль |
|---|---|---|
| Node.js | — | Среда выполнения |
| TypeScript | 5.9.3 | Язык разработки |
| Express | 5.2.1 | HTTP-фреймворк |
| better-sqlite3 | 12.6.2 | SQLite driver (синхронный, WAL) |
| multer | 2.0.2 | Загрузка файлов (disk storage) |
| pdf-parse | 2.4.5 | Извлечение текста и таблиц из PDF |
| xlsx | 0.18.5 | Чтение Excel-файлов (SheetJS) |
| string-similarity | 4.0.4 | Dice-коэффициент для нечёткого сопоставления |
| pdfkit | — | Генерация PDF (подключена, не активно) |
| cors | — | CORS-middleware |
| dotenv | — | Переменные окружения |
| ts-node-dev | — | Hot reload для разработки |

**Тип модулей:** CommonJS (`"type"` не указан, `ts-node` с CJS)

## Frontend

| Технология | Версия | Роль |
|---|---|---|
| React | 19.2.0 | UI-фреймворк |
| TypeScript | 5.9.3 | Язык разработки |
| Vite | 7.2.4 | Сборщик и dev-сервер |
| axios | 1.13.4 | HTTP-клиент |

**Роутинг:** собственный на `useState` (без react-router)
**Стили:** кастомный CSS (`App.css`)

## База данных

| Технология | Роль |
|---|---|
| SQLite (файл) | Единственная БД |
| better-sqlite3 | Синхронный доступ, WAL mode |

**Нет ORM** — запросы пишутся напрямую на SQL через `db.prepare(...).run/get/all`

## Инфраструктура

| Технология | Роль |
|---|---|
| Docker | Контейнеризация |
| Docker Compose | Оркестрация (один сервис) |
| GHCR (GitHub Container Registry) | Хранение образов |

**Продакшн:** VPS `5.42.103.63`, запуск через `docker compose up -d`
