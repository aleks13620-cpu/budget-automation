## Формат таблицы

**Один лист «Эталон». Строка 1 — заголовки. Строки со 2-й — данные.**

| Колонка | Содержимое |
|---|---|
| A | Имя файла (точное, с расширением) |
| B | № строки позиции внутри счёта (1, 2, 3…) |
| C | Наименование позиции **точно как в счёте** |
| D | Родительская группа/раздел (если счёт разбит по разделам, иначе пусто) |
| E | Количество |
| F | Единица измерения (м, шт, кг, комплект…) |
| G | Цена за 1 единицу **С НДС** (правило расчёта ниже) |
| H | Ставка НДС из документа (20%, 0%, `20% дефолт`) |
| I | Как указана цена: `с НДС` / `без НДС+ставка` / `без НДС+сумма НДС` / `неясно` |
| J | Комментарий агента (необычная структура, сомнение в данных и т.д.) |
| K | **ОСТАВИТЬ ПУСТЫМ** — Иван/Сергей: статус (`ОК` / `Исправлено` / `Пропустить`) |
| L | **ОСТАВИТЬ ПУСТЫМ** — Иван/Сергей: правильное наименование (если исправлено) |
| M | **ОСТАВИТЬ ПУСТЫМ** — Иван/Сергей: правильная цена с НДС (если исправлена) |
| N | **ОСТАВИТЬ ПУСТЫМ** — Иван/Сергей: правильное кол-во (если исправлено) |

> Если файл нечитаемый — добавь одну строку с именем файла в A, «нечитаемый» в J, остальное пусто.

## Как считать цену с НДС (колонка G)

Всегда пишем **цену за 1 единицу товара С НДС**.

**Ситуация 1** — есть колонка «Цена с НДС»  
→ берёшь число напрямую, I = `с НДС`

**Ситуация 2** — есть «Цена без НДС» + ставка НДС (например, 20%)  
→ цена_без_НДС × 1.20, I = `без НДС+ставка`  
→ пример: 385 × 1.20 = **462 руб.**

**Ситуация 3** — есть «Цена без НДС» + колонка «Сумма НДС» по строке + количество  
→ (цена_без_НДС × кол-во + сумма_НДС_строки) / кол-во, I = `без НДС+сумма НДС`

**Ситуация 4** — ставка НДС нигде не указана  
→ умножай на 1.20, H = `20% дефолт`, I = `без НДС+ставка`

**Ситуация 5** — документ без НДС (УСН, «НДС не облагается»)  
→ цену берёшь как есть, H = `0% УСН`, I = `с НДС`

> Если сомневаешься — применяй ситуацию 4 и пиши в J: `НДС под вопросом`

## Что считать позицией товара

- Строки с наименованием + ценой/количеством = позиция
- Строки-заголовки разделов (без цены и количества) → записывай в колонку D следующих позиций как родительская группа, сами в таблицу не добавляй
- Итоговые строки («Итого», «Сумма», «НДС итого») → пропускай

## Инструкция для проверяющих (Иван и Сергей)

После того как агент создаст файл, вам нужно проверить каждую строку.

### Что делать

Откройте `benchmark-draft.xlsx`. Для каждой строки найдите оригинальный счёт в папке
`data/uploads/` и сверьтесь с ним.

**Колонка C — наименование**  
Должно совпадать с тем, что написано в счёте. Если агент что-то обрезал или ошибся —
запишите правильное в **колонку L**.

**Колонка G — цена за единицу с НДС** (самое важное поле)  
Нужна цена за ОДНУ единицу, ВКЛЮЧАЯ НДС:
- Есть колонка «Цена с НДС» → это и есть правильное число
- Только «Цена без НДС» + ставка 20% → правильная цена = цена × 1.20 (пример: 385 × 1.20 = **462 руб.**)
- Счёт без НДС (упрощёнка) → цена как есть

Если агент ошибся → запишите правильное число в **колонку M**.

**Колонки E, F — количество и единица измерения**  
Сверьтесь с оригиналом. Если ошибка → запишите правильное кол-во в **колонку N**.

**Колонка K — поставьте статус:**

| Значение | Когда ставить |
|---|---|
| `ОК` | Всё верно, ничего не меняли |
| `Исправлено` | Поправили хотя бы одно поле (L, M или N) |
| `Пропустить` | Строка из нечитаемого файла или явный мусор |

### Чего не нужно делать

- Не нужно разбираться в коде или системе
- Если непонятно как считать НДС по конкретному счёту — напишите в колонке K: `вопрос` и переходите к следующей строке
- Не нужно проверять все 100% строк: если файл явно читаемый и агент справился — можно выборочно проверить 5–10 позиций и поставить `ОК`
