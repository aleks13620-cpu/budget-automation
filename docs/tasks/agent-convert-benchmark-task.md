# Задание для агента: конвертация phase0_audit.xlsx → benchmark-data.json

## Контекст

Проект автоматизации расчётов для строительной компании. Иван и Сергей заполнили
файл `scripts/phase0_audit.xlsx` — **одна строка = один счёт-файл**.
Внутри строки все наименования, цены и количества перечислены через точку с запятой `;`.

Задача агента: прочитать этот файл и развернуть его в `scripts/benchmark-data.json`,
где **одна запись = одна позиция товара**.

**Не трогай исходный Excel-файл. Только читай его и создавай JSON.**

### Структура листа «Фаза 0 — Эталон»

- **Строка 1** — заголовок (объединённые ячейки, пропускаем)
- **Строка 2** — группы столбцов (пропускаем)
- **Строка 3** — заголовки столбцов (пропускаем)
- **Строки 4+** — данные: **одна строка = один счёт**

| Колонка | Содержимое |
|---|---|
| A | № (порядковый номер) |
| B | Имя файла счёта (точное, с расширением) |
| C | Поставщик (название из счёта) |
| D | Кол-во позиций — **сколько товаров реально в счёте** (число) |
| E | Наименования всех позиций — **все через `;`** (или `«ОК»` если без ошибок) |
| F | Статус наименований: `ОК` / `Ошибка` |
| G | Цены за ед. с НДС — **все через `;`** (десятичный знак — запятая: `4067,48`) |
| H | Статус цен: `ОК` / `Обнуление` |
| I | Кол-во и ед.изм. — **все через `;`** (например: `6 шт; 1 шт; 2 шт`) |
| J | Статус кол-ва: `ОК` / `Ошибка` |
| K | Родительский раздел (если счёт разбит на разделы, иначе пусто) |

## Правила разворачивания строк

Для каждой строки данных (начиная с row 4):

1. **Пропустить строку**, если:
   - колонка B (имя файла) пустая, **или**
   - колонки D, E, G, I все пустые (строка не заполнена Иваном/Сергеем).

   На каждый счёт в аудит-файле **ровно одна строка полностью заполнена** — именно её и обрабатываем.
   Остальные строки (пустые шаблонные заготовки без данных) — игнорируем.

2. **Разбить** колонки E, G, I по символу `;` → получаем списки значений.
   - Обрезать пробелы вокруг каждого элемента после split.
   - Если E = `«ОК»` или `ОК` — наименования не искажены, брать оригинальные из счёта
     (в JSON поле `names_ok: true`, имена не раскрываем — они будут взяты из парсера).
   - Пустые элементы после split — пропускать.

3. **Число позиций** (колонка D) — использовать как контрольную проверку.
   Если количество элементов после split в E (или G, или I) ≠ D — записать предупреждение.

4. **Для каждого индекса i** (от 0 до len-1) создать запись:

```json
{
  "file": "имя файла из col B",
  "supplier": "поставщик из col C",
  "position": 1,
  "name": "наименование из col E для данной позиции (null если col E = ОК)",
  "price_with_vat": 4067.48,
  "quantity": 6.0,
  "unit": "шт",
  "section": null
}
```

5. **Цена** (col G[i]): заменить запятую на точку, преобразовать в float.
   Если не получилось → `null`.

6. **Количество и единица** (col I[i]): разбить по пробелу.
   - Первая часть — число (float).
   - Остаток — единица измерения (строка).
   - Пример: `"6 шт"` → quantity=6.0, unit=`"шт"`.
   - Если не получилось → quantity=`null`, unit=`null`.
