# Планы проекта

Эта папка — единая точка хранения всех планов и связанных материалов.

## Структура

- `docs/plans/active/` — активные планы в работе.
- `docs/plans/archive/` — завершенные планы (не удаляются, а архивируются).
- `docs/plans/references/` — вспомогательные материалы: ТЗ, анализ, заметки.
- `docs/plans/reviews/` — периодические ревью процесса (например, раз в 2 недели).
- `docs/IMPLEMENTATION_LOG.md` — краткий журнал того, что реализовано по планам.

## Правила

1. Новый план добавляется в `docs/plans/active/`.
2. После завершения план переносится в `docs/plans/archive/YYYY-MM/`.
3. После переноса в архив в `docs/IMPLEMENTATION_LOG.md` добавляется запись:
   - дата закрытия;
   - название плана;
   - короткий итог;
   - ссылка на архивный файл;
   - опционально коммиты/PR.
4. Планы не удаляются без следа: история всегда сохраняется через архив и Git.
5. Внутренние планы Cursor из `.cursor/plans/` считаются локальными; канон репозитория находится в `docs/plans/`.
6. Ограничение WIP: одновременно допускается только **1** активный технический план в `docs/plans/active/`.
7. До старта реализации в плане обязательно должны быть:
   - ссылка на бизнес-контекст (`.business/INDEX.md` и профильные файлы);
   - явные acceptance criteria фазы;
   - критерий завершения фазы (Definition of Done).

## Недельный операционный ритм

- **Понедельник (Plan):**
  - выбрать одну цель недели;
  - обновить/создать один активный план в `docs/plans/active/`.
- **Среда (Control):**
  - сверить факт выполнения против acceptance criteria;
  - обновить риски и ближайшие шаги в активном плане.
- **Пятница (Close):**
  - закрыть завершенную фазу: проверка -> коммит(ы) -> запись в `docs/IMPLEMENTATION_LOG.md` -> деплой по `AGENTS.md`;
  - если критерии не закрыты, фаза остается открытой.

## Скрипт жизненного цикла

- Добавить новый план в активные:
  - `node scripts/plan-workflow.mjs ingest "C:/path/to/plan.md" --normalize`
- Закрыть активный план и перенести в архив:
  - `node scripts/plan-workflow.mjs complete "docs/plans/active/plan_name.md" --summary "Реализован API;Добавлены тесты" --commits "abc123..def456"`
- Проверка без изменений:
  - `node scripts/plan-workflow.mjs ingest "C:/path/to/plan.md" --dry-run`

## Автоматический контроль незакрытых планов

- Базовая проверка:
  - `cd backend && npm run plans:check`
- Блокирующая проверка (exit code 1 при нарушении WIP/зависших планах/пустом логе):
  - `cd backend && npm run plans:check:strict`
- Гибкая настройка порогов:
  - `node scripts/plan-guard.mjs check --strict --max-active 1 --stale-days 10 --review-days 14`

## Definition of Done для закрытия фазы

Фаза закрывается только при выполнении всей цепочки:

1. Проверка результата (локальная/интеграционная, по контексту фазы).
2. Коммит(ы), покрывающие завершенные acceptance criteria.
3. Запись результата в `docs/IMPLEMENTATION_LOG.md`.
4. Деплой и проверка здоровья по чек-листу из `AGENTS.md`.
