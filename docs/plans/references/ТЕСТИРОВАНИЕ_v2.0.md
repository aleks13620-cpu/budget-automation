# Инструкция по проверке изменений Budget-Automation v2.0

## Подготовка

```bash
# Запустить бэкенд
cd backend && npm run dev

# Запустить фронтенд (в отдельном терминале)
cd frontend && npm run dev
```

Открыть http://localhost:5173

---

## 1. База данных — новые таблицы и колонки

Проверить что миграции прошли:

```bash
node -e "
const db = require('better-sqlite3')('database/budget_automation.db');
// Новые колонки
const cols = db.prepare(\"PRAGMA table_info(specification_items)\").all().map(c => c.name);
console.log('spec_items columns:', cols.filter(c => ['article','product_code','marking','type_size','full_name','parent_item_id'].includes(c)));

const icols = db.prepare(\"PRAGMA table_info(invoices)\").all().map(c => c.name);
console.log('invoices.vat_rate:', icols.includes('vat_rate'));

const scols = db.prepare(\"PRAGMA table_info(specifications)\").all().map(c => c.name);
console.log('specifications.raw_data:', scols.includes('raw_data'));

// Новые таблицы
const tables = db.prepare(\"SELECT name FROM sqlite_master WHERE type='table'\").all().map(t => t.name);
console.log('new tables:', tables.filter(t => ['invoice_items_history','specification_parser_configs','size_synonyms','specification_items_history','spec_parse_rules'].includes(t)));

// Синонимы
const syns = db.prepare('SELECT COUNT(*) as c FROM size_synonyms').get();
console.log('size_synonyms count (должно быть 19):', syns.c);
"
```

**Ожидаемый результат:** все колонки есть, все таблицы созданы, 19 синонимов.

## 2. Загрузка спецификации — новые колонки и raw_data

1. Открыть любой проект → **Загрузить спецификацию** (xlsx с колонками "Артикул", "Производитель", "Типоразмер")
2. После загрузки проверить в БД:

```bash
node -e "
const db = require('better-sqlite3')('database/budget_automation.db');
const items = db.prepare('SELECT id, name, article, type_size, manufacturer, full_name FROM specification_items ORDER BY id DESC LIMIT 5').all();
console.table(items);

const spec = db.prepare('SELECT id, raw_data IS NOT NULL as has_raw FROM specifications ORDER BY id DESC LIMIT 3').all();
console.table(spec);
"
```

**Ожидаемый результат:**
- `article`, `type_size`, `manufacturer` заполнены (если были колонки в файле)
- `raw_data IS NOT NULL = 1` у новых спецификаций

## 3. Объединение многострочных позиций (mergeMultiline)

Загрузить спецификацию где одна позиция разбита на несколько строк, например:
```
Металлическая сетка (ж.с.=0,8)
Воздуховоды прямоугольные
толщиной 6=0,7мм. сеч. 300x200
```

**Ожидаемый результат:** строки "толщиной..." объединяются с предыдущей позицией в одно наименование.

Проверить в редакторе спецификации — открыть **Редактировать** у загруженной спецификации и убедиться что позиций меньше чем строк в Excel.

---

## 4. Строки "То же"

Загрузить спецификацию со строками вида:
```
1 | Клапан шаровой Ду50 PN16 | шт | 5
  | То же, Ду32              | шт | 3
  | То же, Ду25              | шт | 2
```

Проверить в БД:
```bash
node -e "
const db = require('better-sqlite3')('database/budget_automation.db');
const items = db.prepare('SELECT position_number, name, full_name FROM specification_items WHERE name LIKE \"%То же%\" OR full_name LIKE \"%Клапан%\" ORDER BY id DESC LIMIT 10').all();
console.table(items);
"
```

**Ожидаемый результат:**
- Строки "То же" превратились в самодостаточные имена: `"Клапан шаровой Ду50 PN16 Ду32"`, `"Клапан шаровой Ду50 PN16 Ду25"`
- `name` = полное самодостаточное наименование (не "То же, Ду32")

## 5. Редактор спецификации

1. Открыть проект → нажать **Редактировать** у любой спецификации (загруженной после v2.0)
2. Убедиться что открылась страница редактора с таблицей Excel-данных
3. Убедиться что маппинг колонок **авто-определился** (выпадающие списки заполнены)
4. Изменить строку заголовка — таблица подсвечивает выбранную строку синим
5. Изменить маппинг колонки — соответствующая колонка в таблице подсвечивается синим
6. Нажать **Пересобрать позиции** → появится диалог подтверждения → подтвердить
7. Появится **зелёное сообщение с галочкой** "Импортировано X позиций"
8. Нажать **Сохранить конфиг** → "Конфигурация сохранена"
9. Закрыть и снова открыть редактор — маппинг должен восстановиться из сохранённого конфига

**Тест защиты:** очистить поле "Наименование" в маппере → нажать "Пересобрать" → должна быть ошибка "Не выбрана колонка Наименование".

## 6. GigaChat обогащение спецификации

1. Открыть редактор спецификации с загруженными позициями
2. Нажать **Улучшить через GigaChat**
3. Дождаться результата (~2-3 минуты для 100+ позиций)
4. Откроется модальное окно с таблицей diff: **Было / Станет** по полям
5. Убедиться что поле `name` **не изменяется** — только `type_size`, `characteristics`, `article`, `manufacturer`
6. Проверить качество: например "Ду50" → в `type_size` появилось "DN50"

**Тест сохранения в память:**
- Отметить чекбокс **"Сохранить правила в память для обучения"**
- Нажать **Применить изменения**
- Проверить что правила сохранились:

```bash
node -e "
const db = require('better-sqlite3')('database/budget_automation.db');
const rules = db.prepare('SELECT field, raw_value, corrected_value, times_used FROM spec_parse_rules ORDER BY id DESC LIMIT 10').all();
console.table(rules);
"
```

**Тест без сохранения:** применить без галочки → в `spec_parse_rules` ничего не добавляется.

## 7. История версий спецификации + откат

1. В редакторе спецификации нажать **История версий**
2. После применения GigaChat должна появиться запись `gigachat_enrich`
3. Нажать **Откатить** у предыдущей версии → подтвердить
4. Позиции должны вернуться к предыдущему состоянию
5. В истории добавится запись `rollback_to_vX`

---

## 8. НДС в счетах

Загрузить PDF-счёт с таблицей где есть колонки "Сумма без НДС" и "Сумма с НДС".

Проверить в БД:
```bash
node -e "
const db = require('better-sqlite3')('database/budget_automation.db');
const inv = db.prepare('SELECT id, file_name, vat_rate FROM invoices ORDER BY id DESC LIMIT 5').all();
console.table(inv);
"
```

**Ожидаемый результат:** `vat_rate` заполнен (22 по умолчанию, или значение из счёта).

## 9. История версий счёта + откат

1. Открыть любой счёт категории A (Excel) → нажать **История версий**
2. Нажать **Рассчитать цены (сумма ÷ количество)** — создаётся снапшот
3. В истории появится запись `calculate_prices`
4. Нажать **Откатить** у предыдущей версии → цены вернулись

## 10. Рассчитать цены (сумма ÷ количество)

1. Открыть счёт где у позиций заполнена `amount` и `quantity`, но не заполнена `price`
2. Нажать **Рассчитать цены (сумма ÷ количество)**
3. Проверить что `price = amount / quantity` у затронутых позиций

## 11. Синонимы размеров в матчинге (ДУ ↔ DN)

1. Загрузить спецификацию с "Ду50" в наименовании
2. Загрузить счёт с "DN50" в наименовании того же товара
3. Запустить сопоставление
4. **Ожидаемый результат:** позиции сопоставились несмотря на разное написание размера

## 12. Сопоставление по артикулу спецификации

1. Загрузить спецификацию с заполненной колонкой "Артикул" (например, `Q1401250010`)
2. Загрузить счёт где в колонке "Артикул" то же значение
3. Запустить сопоставление
4. **Ожидаемый результат:** confidence ≥ 0.98 (`exact_article`), позиции сопоставлены точно

## 13. Категория C при 0 позициях GigaChat

Загрузить PDF-счёт который GigaChat не смог распознать (пустой результат).

**Ожидаемый результат:** счёт получает категорию `C` (не `B`), предлагается ручной ввод или запрос Excel.

## 14. CategoryB → автопереход в CategoryA после пересборки

1. Открыть счёт категории `B` (PDF с нераспознанными колонками)
2. Выбрать разделитель → **Предпросмотр разделения**
3. Настроить маппинг колонок → **Пересобрать счёт**
4. **Ожидаемый результат:** страница автоматически переходит в CategoryA с загруженными позициями (не остаётся на экране CategoryB)
