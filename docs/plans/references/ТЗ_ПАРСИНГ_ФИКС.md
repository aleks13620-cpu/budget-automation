# ТЗ: Исправление парсинга счетов

## Проблемы

### П1. Парсер берёт таблицу реквизитов вместо товаров
- В "Настройка колонок" dropdown показывает БИК/банк/ИНН вместо товаров
- `items[]` пустой или содержит мусор
- Счета: №2600000046 (Дюкс), Фрегат №111

### П2. Поставщик = мусор
- Колонка "Поставщик" = текст условий/примечаний
- Должно быть: `supplier.name` или `supplier.inn`

### П3. БИК склеен с корсчётом
- `bik` = `04452536030101810445250000360` (БИК+корсчёт)
- Должно: БИК 9 цифр, корсчёт отдельно

### П4. Настройка колонок не работает
- После "Сохранить" нет подтверждения
- После "Пересобрать" превью не обновляется
- В превью: цена = "м", числа в пустых колонках

### П5. Кэш 304 блокирует обновление
- После reparse → GET preview возвращает 304
- UI показывает старые данные

## Решение (минимальное для MVP)

### Этап 1: Фильтр таблиц (6-8 ч)

**Задача:** Исключить реквизитные таблицы из кандидатов

**Реализация:**
```python
def is_requisites_table(table_headers, table_sample_rows):
    """Проверка: это таблица реквизитов?"""
    
    # Негативные маркеры (если найдены → НЕ товарная таблица)
    BAD_KEYWORDS = {
        'бик', 'к/с', 'р/с', 'банк', 'получатель', 'плательщик',
        'инн', 'кпп', 'назначение платежа', 'срок', 'очер'
    }
    
    text = ' '.join(table_headers + table_sample_rows).lower()
    
    # Если >= 3 негативных маркера → это реквизиты
    bad_count = sum(1 for kw in BAD_KEYWORDS if kw in text)
    return bad_count >= 3

def find_positions_table(all_tables):
    """Найти таблицу товаров"""
    
    # Позитивные маркеры
    GOOD_KEYWORDS = {
        'товар', 'работ', 'услуг', 'наименован', 
        'артикул', 'колич', 'цена', 'сумма'
    }
    
    for table in all_tables:
        # Пропустить реквизитные
        if is_requisites_table(table.headers, table.sample_rows):
            continue
        
        # Проверить позитивные маркеры
        text = ' '.join(table.headers).lower()
        good_count = sum(1 for kw in GOOD_KEYWORDS if kw in text)
        
        # Если >= 3 позитивных → берём эту
        if good_count >= 3 and len(table.columns) >= 4:
            return table
    
    return None  # Не найдено
```

**Изменения:**
1. Backend: `parse_invoice()` → применить `find_positions_table()`
2. Если таблица не найдена → вернуть ошибку:
   ```json
   {
     "error": "table_not_found",
     "message": "Таблица товаров не найдена"
   }
   ```
3. UI: показать ошибку, не пытаться отобразить превью

**Результат:**
- Реквизиты не попадают в "Настройка колонок"
- `items[]` содержит реальные товары

### Этап 2: Фикс поставщика и БИК (4-6 ч)

**2.1. Поставщик**

Парсить только из блока "Поставщик:" / "Продавец:" в шапке.

```python
def extract_supplier_name(header_text):
    """Извлечь название поставщика"""
    
    # Паттерны
    patterns = [
        r'Поставщик:\s*([^,\n]+)',
        r'Продавец:\s*([^,\n]+)',
    ]
    
    for pattern in patterns:
        match = re.search(pattern, header_text)
        if match:
            name = match.group(1).strip()
            # Валидация: не должно быть "условия", "самовывоз" и т.п.
            if not any(bad in name.lower() for bad in ['условия', 'самовывоз', 'паспорт']):
                return name
    
    return None  # Не найдено
```

**2.2. БИК**

```python
def normalize_bik_and_cor_account(raw_bik):
    """Разделить БИК и корсчёт"""
    
    if not raw_bik:
        return None, None
    
    # БИК всегда 9 цифр
    match = re.search(r'(\d{9})', raw_bik)
    if match:
        bik = match.group(1)
        # Остаток = корсчёт
        cor_account = raw_bik.replace(bik, '').strip()
        return bik, cor_account if cor_account else None
    
    return None, None
```

**Результат:**
- Поставщик корректный в списке счетов
- БИК = 9 цифр

**3.3. Валидация после reparse**

```python
def validate_parsed_items(items):
    """Проверка качества парсинга"""
    
    errors = []
    
    if len(items) == 0:
        errors.append('Позиции не найдены')
    
    # Проверка price
    valid_prices = sum(1 for item in items if isinstance(item.price, (int, float)))
    if valid_prices / len(items) < 0.7:
        errors.append('Некорректные значения цен')
    
    # Проверка name
    empty_names = sum(1 for item in items if not item.name or len(item.name) < 2)
    if empty_names / len(items) > 0.3:
        errors.append('Пустые наименования')
    
    return errors
```

Если `errors` не пустой → показать warning вместо success.

**Результат:**
- Понятный UI
- Видно, что применилось
- Предупреждения при ошибках
