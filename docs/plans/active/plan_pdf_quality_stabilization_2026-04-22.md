# План: стабилизация качества PDF-парсинга

## Статус
- Текущая фаза: Phase 1 (Baseline + метрики)
- Владелец: Алексей
- Дата старта: 2026-04-22

## Бизнес-контекст (обязательно)
- Основной источник: `.business/INDEX.md`
- KPI: `.business/goals/kpi.md`
- Продукт: `.business/products/overview.md`
- Пользовательский фокус: `.business/audience/avatar.md`

## Цель фазы
Стабилизировать качество распознавания и сопоставления позиций из PDF так, чтобы результат надежно использовался в бюджетировании без ручной переработки большей части данных.

## Scope in
- Валидация входных PDF и обработка невалидных файлов.
- Базовая метрика качества парсинга (точность извлечения строк и ключевых полей).
- Базовая метрика качества сопоставления позиций.
- Контроль качества обучения на свежих правках.

## Scope out
- Полная переработка всех пайплайнов обучения.
- Интеграции следующих этапов (CRM, 1С, мобильное приложение).
- UI-рефакторинг вне необходимых экранов контроля качества.

## Измеримые критерии качества (Acceptance Criteria)
1. На тестовой выборке PDF доля корректно извлеченных строк не ниже 90%.
2. Доля корректного автосопоставления не ниже 80% (минимум из бизнес-KPI), целевой коридор 85%+.
3. После применения обучающих корректировок повторный прогон показывает улучшение метрик минимум на 5 п.п. или удержание на уровне не ниже baseline при расширении выборки.
4. Для каждого сбоя парсинга фиксируется причина (формат/структура/модель) и дальнейшее действие.

## План работ по фазе (3 рабочих дня)
- День 1: зафиксировать baseline на текущей выборке PDF.
- День 2: внедрить точечные улучшения в парсинг/сопоставление и повторить прогон.
- День 3: закрепить рабочую конфигурацию, зафиксировать результаты и решение по следующей фазе.

## Риски и контрмеры
- Риск: разброс качества между типами PDF.
  - Контрмера: раздельная статистика по типам файлов.
- Риск: улучшение на одной выборке ухудшает другую.
  - Контрмера: обязательный повторный прогон на baseline-наборе.
- Риск: потеря фокуса из-за параллельных задач.
  - Контрмера: WIP=1, все идеи вне фазы записывать в backlog/references.

## Definition of Done для фазы
- Все acceptance criteria проверены и имеют зафиксированный результат.
- Подготовлен короткий итог фазы (что улучшилось, что осталось проблемным).
- Изменения закоммичены, и есть готовность к деплою по `AGENTS.md`.
