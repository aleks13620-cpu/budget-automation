- Регрессия более 2 п.п. = откат, поиск причины

- [ ] Заменить: определять VAT rate только из явной колонки VAT rate
- [ ] При отсутствии явной колонки → дефолт 20%, цены не трогать
- [ ] Прогнать benchmark, сравнить с baseline

**Файл:** `backend/src/services/gigachatParser.ts`, функция `mapItems`

### Фаза 2.3 — JSON chunking Вариант B (сжатая схема) | ⏱ 3–4ч | 🤖 Claude | *после 2.1*

**Проблема:** GigaChat-2 обрезает JSON при >30–35 позициях (4096 токенов output,
кириллица ~2 символа/токен). sanitizeJSON восстанавливает структуру, но теряет данные.

**Решение:** сжать JSON-схему ответа (~35% экономия токенов)

- [ ] В промпте: заменить полные имена на короткие:
      `article`→`a`, `name`→`n`, `unit`→`u`, `quantity`→`q`, `price`→`p`
- [ ] В `mapItems()`: добавить маппинг коротких имён обратно в нормальные
- [ ] Убрать из схемы ответа `vat_amount` (дублирует `price × vat_rate`)
- [ ] Прогнать benchmark

**Если документы >60 позиций всё равно обрезаются → переходить к Фазе 2.4.**

**Файл:** `backend/src/services/gigachatParser.ts`, строки 45–200 (промпт), функция `mapItems`
